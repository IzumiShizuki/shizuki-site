【核心组件：水珠感悬浮小按钮】
目标：像一滴半透明水珠贴在玻璃上，轻轻呼吸、带折射与高光，但不花里胡哨。

视觉与材质（CSS 要点）：
- 默认形态：圆形水滴（约 56~64px），半透明，带玻璃质感：
  - 使用 rgba + backdrop-filter: blur(...) + saturate(...)（提供不支持时的 fallback）
  - 细薄描边（半透明白）+ 柔和阴影（不要黑糊一团）
  - 内部有淡淡“齿轮图案”（居中，低对比度、浅色、低透明度），用 inline SVG 或 CSS mask 实现
  - 增加一个很轻的高光（伪元素 radial-gradient），位置略偏上左，表现“水滴顶光”
- 质感关键词：通透、柔、润、轻；避免霓虹、强对比、夸张发光。

动态与液态感（交互重点）：
1) 可拖动（鼠标/触摸都支持）
- 用 Pointer Events（pointerdown/move/up）+ setPointerCapture，移动采用 transform（translate3d）保证丝滑。
- 拖动时水滴要“变形”：像水滴一样被拉扯，而且比较容易形变
  - 根据拖动速度/方向：前进方向圆润稍扁，反向部分尖锐
  - border-radius 也要随之变化（让形变更像液体而不是单纯缩放）
  - 松手瞬间：因为惯性稍微移动一小段距离，（squash & stretch），再稳定成圆润水滴

2) 吸附屏幕边缘
- 松手且滑动结束后如果离得比较近，比如100px则自动吸附到最近的屏幕边缘（左/右/上/下都有，且可以贴到角落）,但是如果大于100px，则保持不动
- 吸附动画要像“水滴被磁力拉过去”：用缓动/弹簧感（requestAnimationFrame 自己写一个简单 spring 或 easeOutBack）
- 吸附后留出安全边距，比方左右露出1/2角落露出1/4，绝不超出屏幕
- 需要考虑窗口 resize：保持相对位置合理，仍贴边

3) 3 秒无鼠标动作自动隐去；移动鼠标再出现
- 监听全局 mousemove / pointermove：重置计时器
- 3 秒没有移动：水滴进入“隐去态”
 - 非吸附态
   - 隐去不要突兀：先轻微向吸附边缘滑入 6~10px + opacity 下降到 0（或接近 0），并禁用 pointer-events
   - 可以留下极淡的“残影”1~2 帧的感觉（很克制，不要闪）
 - 一旦检测到鼠标再次移动：水滴在“上次消失的位置”柔和出现
   - 出现动画：opacity 上升 + 轻微 scale 回弹 + 轻微波纹（极淡）
 - 吸附态
  - 更贴近墙壁/角落，直到融入
 - 一旦检测到鼠标再次移动且之前在隐藏态，水滴从角落中出现
  - 从角落中出现 ：慢慢探出并伴有轻微 scale 回弹 + 轻微波纹（极淡）

4) 待机微动（低调）
- 水滴静止时有非常轻微的“呼吸/浮动”与边缘微波动（幅度很小、周期较长），让它活着但不抢戏
- 尊重 prefers-reduced-motion：如果用户系统要求减少动画，则关闭大部分动效，只保留必要的状态变化（作为变量选择）

【点击展开：小菜单（不喧宾夺主，但很有手感）】
点击水滴后：
- 先“往上一跳”一下（短促、有弹性，像水滴弹起），同时水滴轻微拉长
- 随即水滴往下(不是向左向右向上)变形扩展为一个圆角矩形面板（同材质：同样半透明水感、同样高光与折射）
- 面板里动画延展出 4 个小按钮（按钮风格统一、克制、低饱和）
  - 4 个按钮可以是：关闭(关闭是替代原本的齿轮位，背景色加重，代替原本齿轮位)、对话、记录、设置（图标简洁，SVG）
  - 按钮出现要有层次：依次淡入/上浮 4~8px（很轻）
  - 面板展开要智能：如果本身已经在最下方了稍微往上移动一些(不要快的像瞬移)再展开
- 点击关闭按钮/点击空白处：收起，回到水滴形态（回收动画也要有“液体回缩”的感觉。点击关闭按钮也要有和展开一样的动画）
- 一定要做收起！一定要竖直向下拓展！关闭按钮一定要和原本的开启按钮同位置！

交互细节（学习价值）：
- 拖动时如果菜单是展开状态，也可以拖动菜单
- 菜单展开时不触发“3 秒隐去”
- 点击按钮给一个很轻的水波反馈（缩放 0.96 -> 1.0、加一点点高光闪过即可）
