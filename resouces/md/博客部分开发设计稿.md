# 博客模块阅读体验与演示集成方案 (v0.1)

## 1. 交互与布局架构 (UX & Layout)

本方案旨在为个人博客模块建立兼具专业文档审美与高效沉浸感的阅读环境。核心布局遵循三栏响应式架构：
- **左侧导航 (Left Rail)：** 站点级全局导航与分类索引，支持 `position: sticky` 固定显示。
- **中央正文 (Content Panel)：** 内容主显示区，实施标准化的排版（Typography）约束。
- **右侧能力面板 (Right Sidebar)：** 承载文内目录（TOC）、阅读统计、功能入口（下载/演示）及进度反馈。

---

## 2. 阅读增强特性 (Reader Experience)

### 2.1 文内目录 (TOC) 联动策略
右侧目录组件支持两种动态展开模式，以适应不同长度的内容阅读：
- **全量展开 (Full-Tree Mode)：** 展示完整的 H2/H3/H4 标题树，适用于快速定位。
- **上下文关联 (Context-Only Mode)：** 仅展开当前活动节（Active Section）及其子分支，折叠无关路径，减少视觉干扰。

### 2.2 多重维度阅读反馈
- **元数据统计：** 标题下方展示通过 Markdown AST 解算的“精细化字数统计”、“源文件行数”及基于汉字密度的“预估阅读时长”。
- **环形进度系统：** 右下角按钮集成 SVG 环形进度条，根据 `scrollTop` 实时反馈阅读百分比。

---

## 3. 演示资产集成 (Slidev & PPT Pipeline)

系统支持将文章内容一键转化为演示文稿，实现“内容即演示（Content as Slides）”的工程化闭环。

### 3.1 Slidev 深度整合
- **在线预览：** 基于 iframe 嵌入构建后的 Slidev 静态单页应用（SPA）。
- **自动化转换：** 后端提供基于 Markdown 标题层级的 Slidev Frontmatter 自动填充与 Slide 分隔逻辑。
- **导出能力：** 支持通过管道触发 `slidev export` 任务，下发 PDF 或高保真图片级 PPTX 资产。

### 3.2 资源模型定义
数据层为文章引入 `presentation` 属性集：
- `ppt_url`: 原生 PowerPoint 资产下载链接。
- `slidev_md_url`: 用于二次编辑或版本化的 Slidev 源文件映射。
- `deck_url`: 托管于静态服务器的在线演示地址。

---

## 4. 实施与验收标准 (Implementation & Acceptance)

- [ ] **锚点精中度：** 基于 `IntersectionObserver` 的 TOC 高亮响应偏差需在 10px 阈值内。
- [ ] **语义稳定性：** 阅读进度应基于 `scrollableHeight` 而非固定文本行数计算，确保跨端一致性。
- [ ] **安全隔离：** 预览 iframe 必须启用 `sandbox` 策略，且来源受限于 Content Security Policy (CSP)。
