# 个人学习陪伴网站｜总体设计文档 (v0.2)

## 1. 概述 (Overview)

本文档定义了个人学习陪伴网站的产品边界、技术架构、安全标准及容量规划。旨在为后续开发提供明确的技术指导，确保系统在单机 8GB 内存环境下稳定运行。

---

## 2. 核心结论 (Core Conclusions)

1.  **产品形态：** 采用沉浸式陪伴设计，集成动态角色、背景音效系统及模块化功能组件（Widgets）。
2.  **架构模式：** 采用单体模块化（Monolith Modular）架构，通过模块化解耦便于后续向微服务平滑演进。
3.  **技术栈：** Java 17, Spring Boot 3.x, MyBatis-Plus, MySQL 8, Redis 7, Kafka, OSS。
4.  **AI 配额：** 
    - 基础配额：非自带密钥用户总计限额 5 轮会话。
    - 开发者模式：支持用户配置自有密钥，不受总量限制但受系统限流保护。
5.  **接口规范：** 统一使用 `/api/v1/**` 为 API 根路径。
6.  **安全基线：** 遵循 OWASP ASVS L1 标准，实施强制限流、配额管理及全量审计。
7.  **运行环境：** 目标单机 8GB RAM，通过 Docker Compose 进行容器化编排。

---

## 3. 产品功能架构 (Functional Architecture)

### 3.1 陪伴系统
- **沉浸式交互：** 支持 Live2D/动态角色、环境音混音系统、自定义背景。
- **效率工具：** 通过可配置的浮动组件（Widgets）提供专注计时、任务清单等功能。

### 3.2 内容管理
- **博客系统：** 基于 Markdown 的知识沉淀，支持分类、标签及搜索。
- **应用中心：** 第三方工具集成与快捷访问。

### 3.3 AI 模块
- **兼容性：** 适配 OpenAI 协议。
- **扩展性：** 支持角色卡导入及自定义世界书（Worldbook）。

---

## 4. 用户与权限模型 (Identity & Access)

### 4.1 用户分组 (Groups)
- `GUEST`：游客。
- `USER`：注册用户，具备个人偏好同步能力。
- `FRIEND`：受邀好友，具备更高频次的资源访问权限。
- `ADMIN`：管理员，具备全站治理权限。

### 4.2 权限控制
- 采用 RBAC 模型，权限点精确到 `module.action` 级别。
- 敏感操作实施二级鉴权。

---

## 5. 技术架构设计 (Technical Design)

### 5.1 部署拓扑 (Deployment)

| 组件 | 内存分配 | 角色说明 |
| :--- | :--- | :--- |
| **MySQL 8** | 1.5 GB | 核心持久化存储 |
| **Redis 7** | 0.4 GB | 缓存、会话管理与限流计数 |
| **Kafka** | 0.5 GB | 异步解耦与日志采集 |
| **shizuki-app**| 1.9 GB | 业务单体应用 |

### 5.2 存储策略 (Storage)
- **数据库：** 结构化业务数据。
- **OSS：** 媒体资源文件，支持公开/私有访问隔离，私有资源采用签名 URL 访问。

---

## 6. 安全控制 (Security)

1.  **流量防卫：** 全局请求限流（Rate Limiting）。
2.  **内容安全：** 严格的 XSS 过滤、CSP 策略及外链白名单。
3.  **敏感保护：** 敏感字段（密钥、口令）严禁落入日志，数据库存储采用强哈希算法。

---

## 7. 迭代规划 (Roadmap)

- **Phase 1：** 基础框架搭建、陪伴系统 MVP、偏好同步。
- **Phase 2：** 内容管理（博客、应用中心）上线。
- **Phase 3：** AI 对话与角色系统完善。
