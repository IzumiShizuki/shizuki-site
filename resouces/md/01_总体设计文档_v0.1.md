# 🌌 个人学习陪伴网站｜总体设计：项目星图 (v0.2)

> **导航口径：** 统一产品边界、锚定架构决策、对齐安全基线，为航向 MVP 提供精准指引。  
> **适用航道：** MVP 至初期上线（单机 8GB，Docker Compose 编排）。  
> **决策状态：** 本版为「定型星图」，禁止随意漂移，以防返工风暴。

---

## 🛰️ 0. 核心结论 (The Command Core)

1.  **首屏视窗：** 默认降落在「学习陪伴中心」：注入动态角色 + 沉浸背景 + 混合音效 + 灵动 Dock + 浮动组件（Widgets）。
2.  **动力架构：** 采用 **「核心母舰 + 模块载机」（Monolith Modular）**：`apps/monolith-app` 统筹全局，`modules/*` 与 `libs/*` 精准协同。
3.  **技术引擎：** `Java 17 + Spring Boot 3.x + MyBatis-Plus + MySQL + Redis + Kafka + OSS + Sa-Token`。
4.  **AI 指令配额：** 
    - 游客与无密钥用户：**终身 5 轮**（一问一答 = 1 轮）。
    - 自备密钥（BYOK）：轮数无上限，但受「防过载限流」保护。
5.  **信号波段：** API 路径唯一识别码为 `/api/v1/**`。
6.  **安全屏障：** 严格遵循 OWASP ASVS L1：强制限流、配额管理、白名单阵线、全审计日志与「紧急止损开关」。
7.  **燃料预算：** 锚定单机 8GB 内存。预留 1GB 战略缓冲，确保单体应用与中间件稳健巡航。

## 🎯 1. 产品定位与远景 (Vision & Positioning)

### 1.1 核心坐标
这不仅仅是一个网站，它是你长途跋涉知识荒原时的**全能补给站**。
- **陪伴模式：** 角色之影、背景之韵、环境之息，打造极致专注场域。
- **内容矩阵：** 记录成长的博客、串联时光的时间线、闪耀才华的项目展示。
- **互动星火：** 评论、点赞、反馈，构建有温度的互动。
- **AI 智核：** 角色卡、世界书、记忆长河、OpenAI 兼容大脑。

### 1.2 成功度量衡
- **极速响应：** 游客接入 3 秒内，首屏必须“全量绽放”。
- **意识同步：** 用户偏好（Dock 排序、音效预设、背景）需跨终端实时对齐。
- **指挥权限：** 管理员需在 60 秒内通过「止损开关」控制任一功能模块。

## 🏗️ 2. 用户体系与权限模型 (The Foundation)

### 2.1 身份阶层 (Groups)
- `GUEST`：时空旅者（仅限公开浏览）。
- `USER`：正式公民（基础功能 + 个人偏好记录）。
- `INTERVIEWER`：特邀嘉宾（临时高权，72 小时后自动“能量衰减”）。
- `FRIEND`：挚友阵营（受邀成员，权限优于正式公民）。
- `ADMIN`：星图管理者（最高指挥权）。

### 2.2 契约规则 (ACL)
- **命名规范：** `module.action`（如 `blog.post.create`）。
- **计算逻辑：** 
  - 权限 -> 取「并集」。
  - 配额 -> 取「峰值」。
  - 开关 -> 一处禁用，全域失效（ADMIN 可强制重启）。

## 🎨 3. 信息架构与交互 (The Aesthetic)

### 3.1 学习陪伴页：毛玻璃梦境 (Glassmorphism)
- **视觉风格：** 圆角、半透明、背景模糊的质感，仿佛置身云端。
- **Dock 阵地：** 支持拖拽与吸附，具备记忆功能，是你的“掌上控制台”。
- **Widget 阵列：** 网格化布局，尺寸与配置按需调整。

### 3.2 意识映射 (Data Saving)
- **游客态：** 数据留存于本地 `localStorage`。
- **登录态：** 通过 `/api/v1/me/preferences` 进行云端同步（位置/尺寸除外，保持前端运行时特性）。

---

## 🛠️ 4. 业务功能边界 (Functional Sectors)

### 4.1 应用中心 (App Store)
不仅是工具箱，更是你的个性化外挂。所有外部链接均强制进行「白名单扫描」与「防泄漏重定向」。

### 4.2 博客系统 (Knowledge Repository)
采用 TOAST UI 引擎，追求极简的 Markdown 体验。安全侧落实前端与后端的“双重净化”。

### 4.3 AI 智核 (AI Core)
完全兼容 OpenAI 协议。内置「冷启动保护」，确保在高频请求下核心母舰依然平稳。

## 📦 5. 存储与规划 (Logistics)
