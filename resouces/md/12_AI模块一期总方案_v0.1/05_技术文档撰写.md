## 1. 前端实现口径

### 1.1 路由与页面壳层

保留现有：

1. 路由路径：`/ai-tavern`
2. 顶部菜单 key：`ai-tavern`
3. 顶部快聊按钮：`AI Chat`

升级方向：

1. `AiTavernPage.vue` 从单一聊天布局改为 AI Hub。
2. AI Hub 内部采用标签或分区切换承载六块能力。
3. 快聊侧栏继续沿用当前右侧面板形态，但 UI 文案对齐 AI Hub。

### 1.2 组件拆分建议

建议新增组件分层：

1. `AiHubShell`
   - 统一页头、说明、权限提示、标签切换
2. `AiQuickChatPanel`
   - 快聊与最近会话摘要
3. `AiConversationWorkspace`
   - 普通对话 / 酒馆模式共享对话壳
4. `AiTavernConfigPanel`
   - 角色卡、世界书、场景设定
5. `AiTownExplorer`
   - 小镇地图、房间列表、NPC 展示
6. `AiCompanionHome`
   - 自宅 companion 对话与配置
7. `AiAgentRoadmapCard`
   - Agent 占位说明

## 2. 后端接口口径

### 2.1 会话聊天接口

继续沿用主路径：

1. `POST /api/v1/ai-sessions`
2. `GET /api/v1/ai-sessions`
3. `POST /api/v1/ai-sessions/{session_id}/messages`
4. `GET /api/v1/ai-quotas/me`

新增或扩展字段：

- `CreateAiSessionRequest`
  - `title`
  - `mode`：`quick_chat / normal / tavern / town_npc / companion`
  - `character_id`
  - `worldbook_ids[]`
  - `scene_prompt`
  - `town_room_code`
  - `actor_code`
- `SendAiMessageRequest`
  - `message`
  - `context[]`
  - `memory_enabled`
  - `scope_id`

目标响应：

- `AiSessionSummaryResponse`
- `AiMessageSendResponse`
- `AiQuotaSummaryResponse`

现状中的 `Map<String,Object>` 响应仅视为过渡实现，不再作为稳定公共契约。

### 2.2 角色与世界书接口

3. `POST /api/v1/ai-worldbooks`
4. `GET /api/v1/ai-worldbooks`

## 4. 记忆系统口径

### 4.1 scope 规则

统一格式建议：

`<user_id>:<domain_type>:<actor_code>:<scene_code>`

示例：

1. `1001:town_npc:librarian:library`
2. `1001:companion:my_home_ai:home`

### 4.2 注入顺序

仅当 `memory_enabled=true` 时按以下顺序组装：

1. `profile`
2. 最近有效 `summary`
3. 当前场景相关 `episodic`
4. 最近 `journal`
5. 本轮实时上下文

### 4.3 防串味规则

1. 检索必须同时命中 `scope_id`。
2. 不允许跨 `actor_code` 复用记忆。
3. 小镇 NPC 和记忆 companion 绝不共享记忆域。
4. 公开场景浏览数据不写入记忆。

## 5. 测试与质量门禁

后续实现阶段必须固定执行：

1. `mvn -DskipTests compile`
2. `modules/ai-module` 定向测试
3. 前端相关单测
4. `/ai-tavern` 与顶部菜单手工回归

固定验收场景：

1. `/ai-tavern` 六区块入口与权限提示正确。
2. 顶部 `AI Chat` 可继续快聊。
3. 普通对话与酒馆模式 prompt 装配不同。
4. 酒馆模式可绑定角色卡和世界书。
5. AI 小镇公开可逛，非管理员不可与特殊 NPC 对话。
6. 非管理员不可进入自宅。
7. RPGMaker 文件导入成功与失败路径都可预览反馈。
8. companion 支持人格配置与记忆开关。
9. 关闭记忆后行为退化为普通聊天。
10. 记忆检索不得跨 `scope_id`。
