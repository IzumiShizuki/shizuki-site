# 前端状态与偏好数据规范

## 1. 轻应用偏好主结构

`light_apps` 的 canonical 结构固定如下：

```json
{
  "enabled_codes": ["timeprism-todo"],
  "rail_slots": [
    { "enabled": true, "item_kind": "app", "item_ref": "timeprism-todo" }
  ],
  "collections": [
    {
      "collection_id": "default",
      "title": "集合",
      "items": []
    }
  ],
  "app_configs": {}
}
```

## 2. 字段口径

- `enabled_codes`
  - 已启用轻应用列表
- `rail_slots`
  - 悬浮栏主槽位数组
  - 是当前唯一主持久化结构
- `collections`
  - 悬浮栏集合定义
- `app_configs`
  - 轻应用业务参数

## 3. 历史兼容字段

以下字段只允许用于兼容旧数据，不得继续作为新代码的主语义：

- `ball_slots`
- `ballSlots`
- `floating_slots`
- `floatingSlots`
- `floatingCodes`
- `enabledCodes`

兼容原则：

- 读取时可以接收历史别名
- 归一化后必须回到 `rail_slots`
- 写入偏好时以 canonical state 为准

## 4. 悬浮栏运行时状态

运行时 UI 状态使用前端 `camelCase`：

- `panelKind`
- `activeCollectionId`
- `expanded`
- `dragging`

不再作为正式语义使用：

- `panelMode`
- `panelCollectionId`

## 5. item 字段规范

- `item_kind`
  - `app`
  - `picker`
  - `collection`
  - `url`
- `item_ref`
  - 与 `item_kind` 对应的稳定引用值

## 6. 事件与拖拽来源命名

- 来源标识优先使用 `dock-*`
- `palette-*` 视为历史内部词汇，不再用于新增代码
