# 前端组件分层与命名规范

## 1. 分层

### 路由层

- 使用 `*Page`
- 负责路由参数、页面级数据流、页面级布局切换

### 视图层

- 使用 `*View`
- 负责页面内部的主业务区域

### 轻应用层

- 使用 `*Window`
- 负责可拖拽窗体、独立状态面板、轻应用专属交互

### 公共布局层

- 使用 `*Layout` / `*Shell` / `*Stage`
- 负责区域排布、骨架和容器，不直接承载业务动作

### 公共内容层

- 使用 `*Card` / `*Section` / `*Item`
- 负责重复内容块

### 公共导航层

- 使用 `*Nav` / `*Rail` / `*Sidebar` / `*Dock`
- 负责导航、切换、入口分布

## 2. 当前仓库冻结口径

- `TopMenu.vue`：当前保留为全局菜单的主实现
- `AppMenuBar.vue`：历史别名，包装层文件已移除，不再保留新的同类透传组件
- `RouteDotRail.vue`：导航轨道组件
- `LevitationBall.vue`：悬浮球组件
- `BalanceLedgerWindow.vue` / `TodoWindow.vue` / `ScheduleWindow.vue` / `ProjectsWindow.vue` / `TaskBoardWindow.vue` / `UrlLinksWindow.vue`
  - 明确属于 `Window`，不再改名为 `Panel`

## 3. 首批组件收口

- 公共页头：`PageIntroHeader`
- 公共轨道骨架：`RailScaffold`
- 轻应用顶部工具栏：`LightAppTopToolbar`
- 轻应用折叠内容壳：`CollapsiblePanel`

## 4. 后续优先级

1. 先替换高度重复的模板壳
2. 再统一同类状态字段命名
3. 最后清理遗留包装层与重复样式块

## 5. 命名判定规则

- 能被路由直接加载：`Page`
- 只能作为页面内部主区域：`View`
- 能在轻应用宿主里独立打开：`Window`
- 只负责排版和容器：`Layout` / `Shell` / `Stage`
- 只负责内容复用：`Card` / `Section` / `Item`
