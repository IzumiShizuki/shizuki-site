# L2D 角色与媒体资产后端实施方案 (v0.1)

## 1. 方案目标 (Objectives)

本阶段旨在打通沉浸式陪伴系统的核心资源链路，在后端建立可支持多种媒体形态、多层级可见性控制及安全校验的资源托管能力。

### 1.1 核心支持项
1.  **资产类型映射：** 统一管理 `STATIC_IMAGE`（静态图）、`ANIMATED_IMAGE`（动态图）及 `LIVE2D_PACKAGE`（L2D 资源包）。
2.  **访问控制：** 实现公开角色池、分组可见资源及私有资源的权限隔离。
3.  **合规性校验：** 建立针对 L2D ZIP 包的结构化校验机制，防御安全漏洞（如 Zip Slip）。
4.  **偏好集成：** 建立偏好同步接口，为前端恢复交互状态提供支撑。

## 2. 实施变更总览 (Implementation Overview)

### 2.1 模块职责扩展
- **`media-module`:** 引入资产种类与可见性枚举，新增 `MDA_ASSET_GROUP_ACL`（访问控制列表）与 `MDA_L2D_PACKAGE`（L2D 元数据扩展表）。
- **`monolith-app`:** 在入口层放行公开资源查询接口，并为管理端接口配置对应的权限门禁。
- **`user-module`:** 交付 `GET /api/v1/me/preferences` 接口，支持用户个性化配置的云端拉取。

### 2.2 资产上传与处理流程
```mermaid

    A[获取上传策略] --> B{参数校验}
```

## 3. 存储与数据模型 (Storage & Data Model)

### 3.1 核心表结构变更
- **MDA_ASSET (基表):** 新增 `asset_kind_code`、`visibility_code`、控制首页公开池的 `home_enabled_flag` 以及用于去重校验的 `object_hash`。
- **MDA_ASSET_GROUP_ACL (权限表):** 用于存储资源与授权分组（Groups）的关联关系。
- **MDA_L2D_PACKAGE (扩展表):** 存储 L2D 模型的元数据，包括 Entry JSON 路径、MOC 文件位置、纹理数量等及其校验状态。

### 3.2 ER 关系映射

## 4. 关键接口说明 (API Specifications)

### 4.1 资源访问与检索
- **`GET /api/v1/assets/public/home-roles`:** 拉取首页公开角色池。仅返回已审计通过且开启展示标记的 PUBLIC 资源。
- **`GET /api/v1/assets/{id}/download-url`:** 动态下发资源链接。对于非公开资源，将根据当前用户的权限组及身份进行严格截断，并签发带时效性的私有 URL。

### 4.2 个人偏好管理
- **`GET /api/v1/me/preferences`:** 获取用户启用的 Widgets 清单及对应的个性化配置。
- **设计决策：** Widgets 的实时坐标与尺寸信息仅由前端运行时维护，后端仅负责持久化业务配置（如 Widget 状态、排序及特定参数）。

## 5. 安全与准入规范 (Validation Requirements)

### 5.1 L2D 资源包校验准则
1.  **路径安全：** 严禁出现绝对路径、反斜杠及 `..` 符号，防止目录穿越。
2.  **结构完整性：** ZIP 包内必须包含唯一有效的 `*.model3.json`，且所有引用的资源文件（Moc, Textures, Physics 等）必须在包内实际存在。
3.  **资源约束：** 限制单个 Entry 的大小及文件总数，防止压缩爆炸攻击。

### 5.2 媒体类型一致性校验
- **MIME 匹配：** 在落库前将对请求声明的 MIME 类型与 OSS 存储端的实际类型进行二次对齐，严禁伪造类型上传。
