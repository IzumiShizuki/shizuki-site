# 🛠️ 个人学习陪伴网站｜代码开发：工匠指南 (v0.2)

> **工匠格言：** 优秀的软件不仅仅是运行，它更是一件精雕细琢的工艺品。本手册为你提供构建「陪伴母舰」的精准刻度与锻造规则。

---

## 📏 0. 规矩与方圆 (Standard Constraints)

### 0.1 契约之美 (API Specs)
- **根基路径：** `/api/v1`
- **格律：** 小写字母 + `kebab-case` + 复数名词（严禁动词入侵 URL）。
- **语言：** 后端以 `camelCase` 思考，但对外信号统一映射为 `snake_case`。
- **回声：** 错误必须以 `application/problem+json` 格式标准化输出。
- **踪迹：** 必须在每一个请求头中透传 `X-Request-Id` 与 `X-Trace-Id`。

### 0.2 安全哨塔
遵循 OWASP ASVS L1 铁律。在每一行代码落地前，先问自己：**它是否通过了鉴权、限流、配额与审计的四重考验？**

## 📦 1. 装备清单 (Tech Stack)

### 1.1 后端火力
- **核心心脏：** `Spring Boot 3.x`
- **权限盔甲：** `Sa-Token`
- **数据熔炉：** `MyBatis-Plus` (配合 MySQL 8.x)
- **瞬时记忆：** `Redis 7.x`
- **脉冲传递：** `Kafka`
- **动态图谱：** `springdoc-openapi`

### 1.2 前端视窗
- **框架：** Vue 3 或 React
- **灵魂画法：** Glassmorphism (毛玻璃风格)
- **交互触角：** Interact.js (拖拽) + Gridstack (网格)
- **书写工具：** TOAST UI Editor

## 🏢 2. 营地选址 (Repo Structure)

采用 **Monorepo** 架构，确保所有模块在同一星图下协同工作：
```text
repo-root/
  apps/
    monolith-app/     <-- 指挥塔：请求分发与全局管控
  modules/
    user-module/      <-- 公民馆：账户、身份与偏好
    content-module/   <-- 展览馆：博客、应用与发布
    media-module/     <-- 仓储区：媒体资源与 OSS 适配
    ai-module/        <-- 智核中心：大模型会话与知识库
  libs/
    common-core/      <-- 零件库：基础模型与异常定义
    common-servlet/   <-- Web 套件：安全、审计、限流
    common-integration/ <-- 外交部：对接 OAuth 与 Storage
```

---

## 🏗️ 3. 工坊套件 (Common Libraries)

### 3.1 `common-core` (原子能力)
提供 `ApiResponse`、`ErrorCode` 以及 `BaseEntity` 等最基本的分子结构，不依赖任何具体的 Web 技术。

### 3.2 `common-servlet` (Web 盾牌)
这是我们的防御阵地，聚合了全局异常处理、追踪过滤器、Sa-Token 拦截器、AOP 审计切面以及基于 Redis 的动态限流。

### 3.3 `common-integration` (边境适配)
负责处理 OSS 存储签发与 OAuth2 授权链路，它是我们与外部次元沟通的翻译官。

---

## 🛡️ 4. 模块职责：守卫你的领域

> [!IMPORTANT]
> 每个模块必须严格守卫自己的边界，禁止越权操作其他模块的领域数据。

- **monolith-app**: 它是唯一的门户，负责在任何信号进入业务层之前完成「身份安检」与「流量整形」。
- **user-module**: 掌控用户的第二生命。处理偏好同步、配额策略以及复杂的 OAuth 绑定逻辑。
- **media-module**: 负责将繁重的媒体资产托管至 OSS，并为前端签发具备时效性的「密钥链接」。
- **ai-module**: 赋予网站智慧。它管理着会话历史、世界书知识，并严密监控 AI 算力的消耗。

---

## ⚖️ 5. 数据正义：命名与安全

- **主数据库命名：** `<模块缩写>_<功能>`，如 `USR_ACCOUNT`。
- **默认三维属性：** 每一张表格必须拥有 `id`、`create_time` 与 `update_time`。
- **透明度原则：** 唯一键（AK）、外键（FK）与索引（IX）必须清晰可考。

---

## 🚀 9. 征战顺序 (Execution Order)

1. **奠基：** 父工程与 `libs/*` 阵地建设。
2. **连接：** `monolith-app` 与 `user-module` 联手打通登录中枢。
3. **扩张：** `content/media` 模块建立资源补给线。
4. **觉醒：** `ai-module` 注入交互灵魂。
5. **洗礼：** 端到端联调、压力测试与终极安全扫描。
