# 前端界面审查与组件整合路线（v0.1）

> 文档定位：基于当前 `fronted/vue3-merged` 的真实代码结构，整理界面层重复点、可合并组件与后续改造路线。
> 更新时间：2026-03-12

---

## 0. 审查范围与原则

### 0.1 本次审查范围

- 前端工程：`fronted/vue3-merged`
- 核心入口：`src/App.vue`
- 页面层：`src/pages/**`
- 组件层：`src/components/**`
- 样式层：`src/styles/**`

### 0.2 本次不做的事情

- 不改后端接口
- 不调整路由结构
- 不更换 Vue 3 + Vite 技术栈
- 不引入新的 UI 框架、Tailwind 或设计系统
- 不做视觉重设计，只做结构收口与组件边界优化

### 0.3 审查准则

- 保持现有 `liquid-material / accent / ripple / motion-managed` 风格语言
- 优先合并“重复结构 + 重复交互 + 重复工具逻辑”，不做全站一刀切组件化
- 优先使用组合式组件与局部抽象，不引入过度通用、难维护的大而全组件

## 1. 现状总览

## 1.1 当前前端结构

- 当前是 `Vue 3 + Vite + vue-router` 单应用。
- `src/App.vue` 承担了全局背景、壁纸音视频、L2D、氛围特效、顶部菜单、AI 对话、全局播放器、歌词条、氛围面板、背景选择器等系统级职责。
- 页面层已经出现一定拆分：
  - 管理页拆出 `components/admin/*`
  - 音乐库拆出 `components/music/*` 与 `pages/music/*`
  - 个人中心拆出 `components/profile/*`
  - 轻应用拆出 `components/lightapps/*`
- 但页面骨架、顶部介绍区、卡片组、工具栏、表格行、弹层关闭逻辑、时间/封面格式化仍有较多“局部手写重复”。

## 1.2 已有可复用资产

当前已经具备可继续扩展的公共组件基础，不需要从零搭设计系统：

- `RouteDotRail.vue`：左侧圆点栏导航，已被 `AdminPage`、`ProfilePage`、`AuthorPage` 复用
- `SubtleScrollArea.vue`：统一滚动容器，已被博客、作者页、音乐页等复用
- `ProfileHeroCard.vue`、`ProfileSectionAccordion.vue`、`ProfileSectionItem.vue`：说明页面内部已经开始向组合式组件演进
- `TrackCollectButton.vue`：说明音乐模块局部交互已经有可复用抽象

## 1.3 主要结构信号

### A. 系统级壳层仍然过重

代码证据：

| 文件 | 行数 | 结论 |
|---|---:|---|
| `src/App.vue` | 3414 | 全局系统容器职责过多，已超过单文件可维护边界 |
| `src/components/MusicPlayer.vue` | 1444 | 播放器 UI、状态转换、列表、视觉化控制耦合偏重 |
| `src/components/TopMenu.vue` | 1294 | 主导航、作者入口、用户菜单、菜单态全部耦合 |
| `src/components/AtmospherePanel.vue` | 1098 | 音乐、环境音、效果器三块集中在一个大面板内 |

### B. 页面骨架重复明显

代码证据：

- `page-header` 出现 9 处
- `eyebrow` 出现 19 处
- `RouteDotRail` 已在多个大页面复用，但承载它的整体骨架仍是每页手写

结论：

- 页面层已经有统一意图，但缺少统一的“页头组件 + 铁轨布局组件 + 占位卡组件”。

### C. 模块内部重复已经到可抽象阶段

代码证据：

- `top-toolbar` 出现 12 处
- `panel-collapse` 出现 49 处
- 音乐模块的 `table-head / table-row / track-cover / 时长格式化` 在搜索结果与歌单详情中重复出现
- 博客模块的左侧切换导航在 `BlogListPage.vue` 与 `BlogPage.vue` 中语义高度一致

### D. 工具逻辑已出现重复复制

代码证据：

- `safeCssUrl` 出现 12 处
- `formatTime()` 出现 11 处
- 多个组件都在手写 `document.addEventListener('pointerdown', ...)` 来处理外部点击关闭

结论：

- 共享工具层与交互钩子层已经具备抽离条件。

## 2. 重复点审查表

## 2.1 公共壳层

| 重复位置 | 建议抽象 | 保留风格方式 | 预期收益 | 改造风险 |
|---|---|---|---|---|
| `src/App.vue` 中的背景图、背景视频、L2D、壁纸音视频、场景特效 | `AppBackgroundStage` | 保留现有背景层级、模糊遮罩、首页特效与 `SceneEffectLayer` | 让 `App.vue` 聚焦路由装配，降低全局壳层复杂度 | 中，需要保证首页与非首页的背景切换不回归 |
| `src/App.vue` 中的 AI 面板、歌词条、全局视觉器、氛围面板、背景选择器 | `AppOverlayStack` | 保留现有浮层层级、动效与移动端 sheet 行为 | 统一管理全局浮层、z-index、关闭规则 | 中，涉及较多全局状态与移动端分支 |
| `src/App.vue` 内背景选择器大弹窗 | `BackgroundPickerDialog` | 保留现有 tab、应用范围选择、壁纸获取区与液态玻璃样式 | 将弹层内容与全局壳层解耦，便于单独迭代 | 低到中 |
| `src/components/TopMenu.vue` 的主导航、MENU Hub、作者入口、用户菜单 | `AppMenuBar` | 保留圆角胶囊、圆形按钮、头像入口与状态点风格 | 简化菜单职责，后续可单独维护路由入口与用户菜单 | 低到中 |
| 多处外部点击关闭的 pointerdown 监听 | `useDismissiblePopover` | 不改变弹层外观，只统一关闭行为与清理时机 | 消除重复监听、降低遗漏与泄漏风险 | 低 |

## 2.2 页面骨架

| 重复位置 | 建议抽象 | 保留风格方式 | 预期收益 | 改造风险 |
|---|---|---|---|---|
| `AdminPage.vue`、`AiTavernPage.vue`、`AuthCallbackPage.vue` 以及后续同类页头 | `PageIntroHeader` | 保留 `eyebrow + h1 + 描述 + hint` 的现有视觉语义 | 减少重复页头模板与 scoped 样式复制 | 低 |
| `AdminPage.vue`、`ProfilePage.vue`、`AuthorPage.vue` 的左侧圆点栏 + 主内容骨架 | `RailScaffold` | 保留 `RouteDotRail + content-panel` 的当前布局与毛玻璃面板 | 统一三类页面的骨架容器与响应式断点 | 中 |
| 管理页 KPI 卡、个人页 overview card 等轻统计卡 | `MetricCardGrid` | 保留玻璃卡片、数字突出、弱文案说明样式 | 减少重复卡片网格与统计卡样式定义 | 低 |
| `ProfilePage.vue` 各组页头 | `SectionGroupHeader` | 保留 group eyebrow、标题、caption 结构 | 让个人中心内部结构更整齐，便于后续扩展 | 低 |
| `ProfilePage.vue` 中多个占位提示块 | `EmptyStateCard` | 保留当前 placeholder card 的轻提示语义 | 把“未实现/暂无内容/后续扩展”统一收口 | 低 |

## 2.3 模块内部

| 重复位置 | 建议抽象 | 保留风格方式 | 预期收益 | 改造风险 |
|---|---|---|---|---|
| `BlogListPage.vue` 与 `BlogPage.vue` 的左侧模式切换导航 | `BlogRouteSidebar` | 保留博客模块现有按钮、提示文案和侧栏排布 | 让博客列表/详情/编辑共享导航壳层 | 中 |
| `MusicLibraryHomeView.vue` 与 `MusicPlaylistDetailView.vue` 的歌曲表格 | `MusicTrackTable` | 保留当前表头、行 hover、封面、操作按钮与收藏按钮风格 | 统一歌曲表格结构，减少表格行与操作列重复 | 中 |
| `BalanceLedgerWindow.vue`、`TodoWindow.vue`、`ProjectsWindow.vue`、`ScheduleWindow.vue`、`TaskBoardWindow.vue`、`UrlLinksWindow.vue` 的顶部工具栏 | `LightAppTopToolbar` | 保留 icon button、toolbar-hint、轻应用暗色玻璃面板风格 | 统一轻应用顶部控制区语义与样式 | 中 |
| 轻应用窗口内大量 `Transition name="panel-collapse"` + 可收起块 | `CollapsiblePanel` | 保留 `panel-collapse` 动效与标题区布局 | 消除折叠面板模板与动效重复 | 中 |

## 2.4 工具函数与样式

| 重复位置 | 建议抽象 | 保留风格方式 | 预期收益 | 改造风险 |
|---|---|---|---|---|
| 多处时间格式化逻辑 | `formatMediaTime` | 不改输出格式，统一 `mm:ss` 规则 | 避免播放器、音乐页、歌词时间各写一套 | 低 |
| 多处 `safeCssUrl` 使用和个别私有实现 | 统一复用现有 `safeCssUrl` | 不改封面与背景使用方式 | 避免 URL 转义逻辑漂移 | 低 |
| 多页重复的 `page-header / eyebrow / hero-btn / switch-btn / top-toolbar` 样式片段 | 通过组件内基类或模块级样式文件统一，而不是全局裸类泛滥 | 保留当前玻璃态、强调色与渐变文本 | 降低 scoped 样式复制，避免同名类多处漂移 | 中 |

## 3. 高优先级合并方案

## 3.1 P0：系统级壳层拆壳

目标文件：

- `src/App.vue`
- `src/components/TopMenu.vue`
- `src/components/MusicPlayer.vue`
- `src/components/AtmospherePanel.vue`

本阶段冻结的抽象目标：

- `AppBackgroundStage`
  - 负责背景图、背景视频、L2D、壁纸音视频、场景特效
- `AppOverlayStack`
  - 负责 AI 对话、歌词条、视觉器、氛围面板、背景选择器
- `BackgroundPickerDialog`
  - 负责背景设置弹窗内部全部 UI
- `AppMenuBar`
  - 负责主路由入口、作者入口、用户菜单、MENU Hub
- `useDismissiblePopover`
  - 负责外部点击关闭、Esc 关闭、事件监听清理

P0 采用原则：

- 只拆职责，不改路由与功能
- 先拆系统壳层，不碰业务接口
- 每次替换必须保持首页、音乐页、移动端面板行为一致

## 3.2 P1：页面骨架统一

优先页面：

- `AdminPage.vue`
- `ProfilePage.vue`
- `AuthorPage.vue`
- `AiTavernPage.vue`
- `AuthCallbackPage.vue`
- 后续可扩展到 `AuthPage.vue`

本阶段冻结的抽象目标：

- `PageIntroHeader`
  - props 语义固定为：`eyebrow / title / description / hint / actions`
- `RailScaffold`
  - slots 固定为：`rail / header / body`
- `MetricCardGrid`
  - 接收统计项数组，输出统一卡片网格
- `SectionGroupHeader`
  - 统一“组眉 + 标题 + 描述”
- `EmptyStateCard`
  - 统一空状态、占位状态、后续扩展提示

采用范围固定：

- `AdminPage`、`ProfilePage`、`AuthorPage` 统一收口到 `RailScaffold`
- `AiTavernPage`、`AuthCallbackPage`、`AdminPage` 的页头统一收口到 `PageIntroHeader`

## 3.3 P1：模块内收口，不强行全站通用

博客模块：

- `BlogListPage.vue` 与 `BlogPage.vue` 统一使用 `BlogRouteSidebar`
- 不把博客侧栏直接提升为全站组件，避免抽象层级过高

音乐模块：

- `MusicLibraryHomeView.vue` 与 `MusicPlaylistDetailView.vue` 统一使用 `MusicTrackTable`
- `TrackCollectButton.vue` 继续作为表格行内操作组件复用
- `formatMediaTime` 与 `safeCssUrl` 进入共享层后，页面不再保留私有实现

轻应用模块：

- `BalanceLedgerWindow`、`TodoWindow`、`ProjectsWindow`、`ScheduleWindow`、`TaskBoardWindow`、`UrlLinksWindow` 统一使用 `LightAppTopToolbar + CollapsiblePanel`
- 保留各自业务表单、列表与状态模型，不做轻应用统一大基类

## 3.4 P2：工具函数、样式令牌、交互钩子统一

- 把重复的 `formatTime()` 合并到 `formatMediaTime`
- 统一复用 `safeCssUrl`
- 多个组件的外部点击关闭统一到 `useDismissiblePopover`
- 模块内重复的 `panel-collapse / top-toolbar / page-header` 样式片段统一为组件基类或模块级样式片段

## 4. 分阶段落地路线

### Phase 1：系统壳层收口

- 拆分 `App.vue` 的背景层、浮层层、背景选择器
- 收口 `TopMenu.vue` 的菜单职责
- 为全局浮层引入统一关闭钩子

交付标准：

- `App.vue` 只保留路由装配与少量全局状态协调
- 系统级弹层关闭规则统一

### Phase 2：页面骨架统一

- 为 Admin / Profile / Author 建立统一 `RailScaffold`
- 为 AI / AuthCallback / Admin 建立统一 `PageIntroHeader`
- 收口轻统计卡与占位卡

交付标准：

- 大页面骨架结构一致
- 圆点栏、页头、占位块不再各写一套

### Phase 3：模块内组件收口

- 博客侧栏抽象为 `BlogRouteSidebar`
- 音乐表格抽象为 `MusicTrackTable`
- 轻应用窗口顶部抽象为 `LightAppTopToolbar`
- 可折叠面板抽象为 `CollapsiblePanel`

交付标准：

- 博客、音乐、轻应用内部的高频重复模板显著下降
- 不把模块内抽象硬提升为全站通用

### Phase 4：工具与样式统一

- 引入 `formatMediaTime`
- 统一 `safeCssUrl`
- 引入 `useDismissiblePopover`
- 清理重复 scoped 样式片段

交付标准：

- 不再出现同一类时间/封面/外部点击关闭逻辑的多份实现

### Phase 对应 beads

- Phase 1：`shizuki-site-5el`
- Phase 2：`shizuki-site-bbo`
- Phase 3：`shizuki-site-ffw`
- Phase 4：`shizuki-site-do2`

## 6. 结论

当前前端不缺视觉风格，也不缺局部优秀组件，真正欠缺的是“系统壳层拆壳、页面骨架统一、模块内重复收口、工具函数归位”这四步。

因此后续实施顺序必须固定为：

1. 先拆系统壳层
2. 再统一页面骨架
3. 再做博客 / 音乐 / 轻应用的模块内收口
4. 最后统一工具函数、交互钩子与样式片段

本路线默认作为下一轮前端重构的设计基线使用。
