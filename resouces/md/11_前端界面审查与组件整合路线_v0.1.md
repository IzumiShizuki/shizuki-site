# 前端界面审查与组件重构方案 (v0.1)

## 1. 深度审查与原则 (Review Principles)

基于当前 `frontend/vue3-merged` 的代码现状，本项目急需通过结构化重构来解决系统壳层过重、组件碎裂化以及交互逻辑冗余等核心问题。

### 1.1 审查准则
- **视觉一致性：** 强制沿用现有的 `Liquid-Material` 与 `Motion-Managed` 风格语言。
- **职责解耦：** 优先对全局容器 `App.vue` 进行拆壳，实行关注点分离。
- **模块化收口：** 针对高频重复的页面骨架、工具函数及交互钩子建立统一的抽象层。

---

## 2. 现状分析与技术债评估 (Current State)

### 2.1 系统级壳层冗余
`src/App.vue` 当前承担了背景舞台、L2D 交互、UI 浮层堆栈及全局状态协调等过多职责，代码量已达 3k+ 行，严重超出单文件维护阈值。

### 2.2 页面骨架一致性缺失
管理后台（Admin）、个人中心（Profile）及创作空间（Author）虽然在视觉上趋同，但其底层骨架模板仍处于“每页手写”状态，缺乏统一的 `Scaffold` 抽象。

### 2.3 逻辑拷贝现象
- **工具流：** 时间格式化（`formatTime`）与样式安全处理（`safeCssUrl`）在各业务模块中存在多份副本。
- **交互流：** 弹层外部点击自动关闭的 `pointerdown` 监听逻辑在数十个组件中重复实现。

---

## 3. 重构路径图 (Refactoring Roadmap)

### 3.1 P0: 系统壳层解耦 (Shell Decoupling)
| 抽象目标 | 职能定义 | 预期收益 |
| :--- | :--- | :--- |
| **AppBackgroundStage** | 托管背景图、视频及 L2D 动态层 | 净化 `App.vue` 模板，隔离背景渲染逻辑 |
| **AppOverlayStack** | 统一管理全局 Sheet、对话框及氛围面板 | 规范 z-index 堆栈与遮罩交互逻辑 |
| **AppMenuBar** | 封装主导航、账户入口及系统菜单 | 简化菜单职责，提升导航入口的可维护性 |

### 3.2 P1: 页面骨架模版化 (Scaffolding)
- **RailScaffold:** 统一 Admin/Profile/Author 页面的侧边圆点导航与主面板布局。
- **PageIntroHeader:** 建立标准化的页眉契约，包含 `eyebrow`、`title`、`description` 等元数据展示。
- **MetricCardGrid:** 抽象管理端与个人中心的统计指标卡片。

### 3.3 P2: 共通工具与组合式函数 (Composables)
- **useDismissiblePopover:** 封装 Esc 关闭、外部点击自动回收的标准化 Hook。
- **formatMediaTime:** 统一媒体时长格式化逻辑，替换各页面私有实现。

---

## 4. 实施阶段与标准 (Phased Execution)

### Phase 1: 核心容器重塑
- 拆分 `App.vue` 至 `BackgroundStage` 与 `OverlayStack`。
- 交付标准：`App.vue` 代码量下降 50% 以上，全站背景切换无视差回归。

### Phase 2: 业务模块组件收口
- 音乐模块：抽象 `MusicTrackTable` 解决各视角下曲目列表的不一致。
- 博客模块：统一 `BlogRouteSidebar` 侧栏。
- 交付标准：对应模块内模板重复率降低 40%。

---

## 5. 验收与回归清单 (Verification)

- [ ] **多端适配：** 确保 PC 与 Mobile 在组件重构后的断点行为一致。
- [ ] **交互对齐：** 验证 Popover 与 Modal 的关闭行为是否遵循统一的全局策略。
- [ ] **性能监控：** 评估组件抽离后是否因 Props 层级加深导致非必要的重绘（Re-render）。

> [!IMPORTANT]
> **结论：** 前期重构必须遵循“先拆壳层、再定骨架、后理细节”的顺序。本方案将作为后续前端开发流程的设计基准。
