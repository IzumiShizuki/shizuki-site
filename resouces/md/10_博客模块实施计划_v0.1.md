# 博客模块实施方案 (v0.1)

## 1. 方案目标 (Objectives)

本阶段旨在建立一套具备高度协作弹性与深度的内容发布体系，涵盖从 Markdown 智能编排、多级权限控制到沉浸式阅读体验的全链路功能。

### 1.1 核心功能范畴
1.  **分层阅读体验：** 提供支持 TOC（目录树）、阅读进度感知、Markdown 原始文件导出的沉浸式阅读器。
2.  **发布合规流程：** 支持草稿态、发布态及下线态的生命周期管理，集成 Markdown 资源中转托管。
3.  **多级可见性体系：** 实现基于 `PUBLIC / PRIVATE / GROUP` 的三级可见性模型，支持按分类策略进行批量权限覆盖。
4.  **智能创作增强：** 前端集成疑似 Markdown 的剪贴板智能识别与格式记忆选择机制。

---

## 2. 技术契约与数据模型 (Technical Specification)

### 2.1 核心 DTO 定义
- **PostSummary:** 列表聚合模型，仅暴露标题、摘要、可见性及阅读时长估算。
- **PostDetailResponse:** 完整详情模型，包含 Markdown 正文、Slug 语义化编码、词数统计及点赞等社交元数据。

### 2.2 数据模型扩展 (CTN_POST)
- **基表扩展:** 新增发文状态、分类编码、语义化别名（Slug）、字数统计及发布时间戳。
- **关联体系:** 引入 `CTN_POST_TAG`（动态标签）与 `CTN_POST_CATEGORY_POLICY`（分类级别的 RBAC 策略）。

---

## 3. 访问控制逻辑 (Visibility & RBAC)

### 3.1 鉴权优先级判定
系统对文章的访问请求遵循以下降级逻辑：
1.  **管理权限校验：** `ADMIN` 角色强制放行。
2.  **文章级重写：** 若文章可见性设为 `PRIVATE` 或 `GROUP`，则忽略分类策略，直接执行文章级 ACL 校验。
3.  **分类策略匹配：** 对于 `PUBLIC` 文章，若其分类（如 `Game` 类别）配置了特定的分组访问限制，则执行对应的分组确认。
4.  **公开兜底：** 无上述限制的文章允许匿名访问。

### 3.2 功能权限点 (Privileges)
- `blog.post.write`: 创作权限。
- `blog.post.publish`: 发布与策略调整权限。
- 初期默认授予 `ADMIN` 与 `FRIEND` 角色。

---

## 4. 专项技术：Markdown 智能识别 (Frontend Strategy)

### 4.1 探测算法
前端监听 `paste` 事件，针对 `text/plain` 内容进行特征扫描，满足以下 2 项及以上特征即判定为“疑似 Markdown”：
- 级联标题（`# `）
- 列表标记（`- `、`* `、`1. `）
- Fenced Code Blocks (``````)
- Markdown 链接/表格/引用标记

### 4.2 交互闭环
1.  **记忆化弹窗：** 首次检测到特征时询问用户处理策略。
2.  **会话级记忆：** 记忆该会话内的选择（按原文插入或转义插入），后续粘贴自动跟随，无需重复弹窗。
3.  **手动重置：** 提供清理会话缓存的入口，恢复初始判别状态。

## 5. 验收方案 (Acceptance Criteria)

- [x] **鉴权隔离：** 确认 `Game` 分类文章对访客（Guest）的不可见性及向授权组的准入性。
- [x] **内容完整：** 验证详情接口及 Markdown 下载接口的字符编码一致性。
- [x] **流程闭环：** 覆盖从“存为草稿”到“公开发布”并最终“下线归档”的全生命周期。
