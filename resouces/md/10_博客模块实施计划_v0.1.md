# 10_博客模块实施计划_v0.1

- 文档日期：2026-03-03
- 适用仓库：`shizuki-site`
- 一期范围：博客阅读链路 + 发文链路 + 分类访问策略 + 粘贴 Markdown 智能识别

---

## 1. 冻结边界与业务规则

### 1.1 一期目标（冻结）

1. 阅读页能力：文章列表、详情阅读、右侧 TOC、TOC 模式切换（全部展开 / 当前小节）、阅读进度环、Markdown 下载。
2. 作者能力：草稿保存、发布、下线、Markdown relay 上传（后端中转 OSS）。
3. 访问控制：文章可见性 `PUBLIC / PRIVATE / GROUP` + 分类默认访问策略 + 文章级覆盖。
4. 搜索口径：仅标题、摘要、标签。
5. 粘贴体验：编辑器粘贴疑似 Markdown 时询问是否按 Markdown 插入，支持会话内记忆选择。

### 1.2 一期不做（冻结）

1. 正文全文检索。
2. 评论系统。
3. Slidev/PPT 的自动化生成和在线预览（仅保留接口位与数据位）。

## 3. 可见性判定顺序与权限码

### 3.1 判定顺序（固定）

1. `ADMIN` 直接放行。
2. 若文章 `visibility=PRIVATE/GROUP`：按文章级规则判定（覆盖分类策略）。
3. 若文章 `visibility=PUBLIC`：命中分类策略时，按分类允许分组二次校验。
4. 分类策略未配置或 `enabled=false`：PUBLIC 文章按公开规则放行。

### 3.2 game 分类默认策略（一期默认）

- 分类：`game`
- 允许分组：`USER / FRIEND / ADMIN`
- 拒绝分组：`GUEST / INTERVIEWER`
- 管理员后续可通过策略接口开放。

### 3.3 发文权限码

- `blog.post.write`
- `blog.post.publish`

默认授予：`ADMIN / FRIEND / INTERVIEWER`（可在管理侧动态调整）。

## 4. 粘贴 Markdown 智能识别（前端）

### 4.1 触发条件

监听编辑区 `paste`，提取 `text/plain`，命中以下特征至少 2 项则判定为“疑似 Markdown”：

1. 标题：`# `
2. 列表：`- `、`* `、`1. `
3. 代码块：``````
4. 链接：`[text](url)`
5. 表格：`|---|`
6. 引用：`> `

### 4.2 交互流程

1. 若会话内无记忆选择：弹窗询问  
   `检测到疑似 Markdown，是否按 Markdown 插入？`
2. 选择“是”：
   - 按 Markdown 原文插入编辑器。
   - 记忆本次会话策略为 `markdown`。
3. 选择“否”：
   - 对 `# * _ \` [] () > - + ! |` 等字符做转义后插入。
   - 记忆本次会话策略为 `plain`。
4. 后续同会话粘贴：
   - 按记忆策略直接执行，不重复弹窗。
5. 提供“重置粘贴判断”按钮，可清空会话记忆。
